# <a name="use-actions"></a>ใช้การดำเนินการ

การดำเนินการเปิดช่วงความเป็นไปได้สำหรับการเขียนตรรกะทางธุรกิจ ด้วยการดำเนินการนี้ คุณสามารถดำเนินการต่าง ๆ เช่น สร้าง อัปเดต ลบ กำหนด หรือดำเนินการกระทำ การดำเนินการสร้างข้อความแบบกำหนดเองภายใน นักพัฒนาพูดถึงการดำเนินการเหล่านี้ว่าเป็น "ข้อความ" แต่ละข้อความเหล่านี้จะยึดตามการดำเนินการที่ทำกับเรกคอร์ดเอนทิตี ถ้าเป้าหมายของกระบวนการคือเพื่อสร้างเรกคอร์ด แล้วอัปเดต และจากนั้นกำหนด มีสามขั้นตอนที่แยกต่างหาก แต่ละขั้นตอนถูกกำหนดโดยความสามารถของเอนทิตี – ไม่จำเป็นว่าต้องเป็นกระบวนการทางธุรกิจของคุณ  
  
การดำเนินการให้ความสามารถในการกำหนดคำกริยาเดียว (หรือข้อความ) ที่ตรงกับการดำเนินการที่คุณต้องการใช้สำหรับธุรกิจของคุณ ข้อความใหม่เหล่านี้จะควบคุมโดยขั้นตอนหรือลักษณะการทำงาน แทนที่สิ่งจะสามารถทำอะไรได้กับเอนทิตี ข้อความเหล่านี้สามารถสอดคล้องกับคำกริยาเช่น เลื่อนระดับ แปลง จัดกำหนดการ กำหนดเส้นทาง หรืออนุมัติ – อะไรก็ตามที่คุณต้องการ การเพิ่มคำกริยาเหล่านี้ช่วยให้คุณมีคำศัพท์มากขึ้น ในการกำหนดกระบวนการทางธุรกิจของคุณอย่างราบรื่น คุณสามารถใช้คำศัพท์ที่มากขึ้นนี้จากไคลเอ็นต์หรือการรวม แทนที่การที่ต้องเขียนการดำเนินการภายในไคลเอ็นต์ นอกจากนี้ยังทำให้ง่ายขึ้นอีกด้วยเนื่องจากคุณสามารถจัดการและบันทึกความสำเร็จหรือความล้มเหลวของการดำเนินการทั้งหมดให้เป็นหน่วยเดียว  
  
<a name="BKMK_ConfigurableMessages"></a>   
## <a name="configurable-messages"></a>ข้อความที่สามารถกำหนดค่าได้  
 เมื่อการดำเนินการถูกกำหนดและเปิดใช้งาน นักพัฒนาสามารถใช้ข้อความนั้นเหมือนข้อความอื่น ๆ ที่ได้มาจากแพลตฟอร์ม อย่างไรก็ตาม ความแตกต่างที่สำคัญคือตอนนี้บุคคลอื่นที่ไม่ได้เป็นนักพัฒนาสามารถทำการเปลี่ยนแปลงกับสิ่งที่ควรทำเมื่อใช้ข้อความนั้น คุณสามารถกำหนดค่าการดำเนินการเพื่อปรับเปลี่ยนขั้นตอนตามการเปลี่ยนแปลงในกระบวนการธุรกิจของคุณ รหัสแบบกำหนดเองใด ๆ ที่ใช้ข้อความนั้นไม่จำเป็นต้องเปลี่ยนตราบเท่าที่อาร์กิวเมนต์กระบวนการไม่เปลี่ยนแปลง  
  
 กระบวนการเวิร์กโฟลว์และปลั๊กอินให้ความสามารถที่คล้ายกันต่อไปสำหรับการกำหนดอัตโนมัติ กระบวนการเวิร์กโฟลว์ยังคงมีความสามารถสำหรับผู้ที่ไม่ใช่นักพัฒนาในการทำการเปลี่ยนแปลง แต่มีความแตกต่างในวิธีประกอบกระบวนการทางธุรกิจและวิธีที่นักพัฒนาสามารถเขียนรหัสของพวกเขา การดำเนินการคือ ข้อความที่ทำงานในระดับเดียวกันเป็นข้อความที่ได้มาจากแพลตฟอร์ม นักพัฒนาสามารถลงทะเบียนปลั๊กอินสำหรับการดำเนินการ

## <a name="global-messages"></a>ข้อความส่วนกลาง 
 
 ไม่เหมือนกับ CDS สำหรับเวิร์กโฟลว์หรือ[ปลั๊กอิน](/powerapps/developer/common-data-service/apply-business-logic-with-code?branch=master#create-a-plug-in)ของแอป การดำเนินการไม่จำเป็นต้องเกี่ยวข้องกับเอนทิตีเฉพาะ คุณสามารถกำหนดการดำเนินการ "ส่วนกลาง" ที่สามารถเรียกใช้ด้วยตัวเอง
